# Capstone Project of Digital Twin Course 
## Line-Following Robot with VSI

This repository contains the Capstone Project for the Siemens Digital Twin Course. It implements a line-following robot simulated using three components (Simulator, Controller, Visualizer) interconnected via **Virtual System Integration (VSI)**.

## 📂 Project Structure

```text
LineFollowingRobot/
├── src/
│   ├── simulator.py         # Subscribes to v, omega (Velocity). Publishes x, y, theta. Adds noise/disturbances.
│   ├── controller.py        # Subscribes to x, y, theta. Calculates PID response. Publishes v, omega.
│   └── visualizer.py        # Subscribes to x, y. Live plots the robot trajectory against the path. Dumps metrics.
├── vsi/
│   ├── build_digital_twin.cmd # Innexis Digital Twin Builder commands. Maps CAN IDs to python scripts.
│   └── LineFollowingDT/     # Auto-generated by vsiBuild. Contains the VSI interconnect routing configuration.
├── run_experiments.sh       # Bash script to automatically spawn the DT and run the 4 mandatory testing scenarios.
└── README.md                # This file.
```

## ⚙️ How to Setup and Run the Simulation

### Step 1: Initialize VSI Environment Paths (VPC)
Since you are using an **Innexis Virtual Private Cloud (VPC)**, you must first connect to your configured VPC instance.
Ensure you have started your Innexis environment and VSI tools are sourced into your `PATH`. For Innexis VPCs, this typically means the `PAVE360VSI_HOME` is already set or you need to load the module/script provided by your lab. You should be able to run `vsiBuild` and `vsiRun` directly from the terminal without errors.

### Step 2: Build the Digital Twin
The `run_experiments.sh` script handles building the DT automatically, but if you want to build it manually:

```bash
cd /vsi/ # go location of the vsi directory

source env_vsi.bash # activate your vsi 

vsiBuild -f build_digital_twin.cmd # build your dtdtl file, define how the system structure and signals

```
This generates the `LineFollowingDT` folder.

### Step 3: Run the Experiments Suite
To execute all mandatory experiments (E1-E4) end-to-end (including the PID Gain Sweeps, Noise Rejection, and Curved Paths), simply run the bash automation script:

```bash
cd /home/adham/Documents/Capstone_project/LineFollowingRobot/
chmod +x run_experiments.sh
./run_experiments.sh
```

> **Note**: This script automatically spawns the VSI server, spins up the Controller, Simulator, and Visualizer, safely closes them upon run completion, and repeats for every test.

## 📊 Viewing the Results

After `run_experiments.sh` completes successfully:
1. **Plots:** `LineFollowingRobot/output/plots/*.png`
2. **KPI Metrics:** `LineFollowingRobot/output/results/results.csv`

### Analyze Results

```bash
python3 experiments/analyze_results.py --csv output/results/results.csv
```
